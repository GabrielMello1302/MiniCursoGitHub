1º: O que é o Git?

      O Git é um versionador de códigos, ou seja, ele cria versões do código.
        "Por exemplo uma empresa aonde tenha três programadores, todos trabalhando no mesmo sistema e
    cada um em uma determinada tarefa. o P1 trabalhando na Tela-Login, P2 na Tela-Informaçoes e P3 na Exclusão-de-Dados."

      O Git torna-se possivel que cada um trabalhe em uma versão sem atrapalhar o outro, com a possibilidade de ser versões 
    diferentes ou a mesma. Para que isso funcione cada programador teria que ter em sua maquina uma versão do código.

      Para que não seja necessário ficar copiando e colando o código um do outro, o Git chega para ajudar, com o auxilio
    de alguns comandos é possivel armazenar tudo dentro de uma "Nuvem" que fica salvo na internet.

            "Por exemplo o P1 quer trabalhar no código, ele vai no repositório do código, baixa ele e cria uma ramificação na 
    sua própria maquina, o P2 e P3 fazem a mesma coisa. Caso o P2 termine seu trabalho ele vai pegar a versão dele jogar no servidor que esta         
    sendo armazenado o código o P3 também." 
      
      Caso o P1 tenha subido o código dele primeiro e o P2 logo após e os dois tenham alterado alguma coisa igual, o Git faz todo esse controle       
      para nós, todo esse versionamento e criação de versões ele faz através de comandos e programas que tenham interface.

2º: O que é o GitHub?

      Ele é o repositório (Caixinha) onde armazenamos todos esses códigos, projetos, etc. Ele é o mais conhecido e dentro dele é possivel fazer a criação de varios repositórios.
      Onde o repositório armazena todo o cógido, e pra cada alteração feita, é salvo com um comentário dentro desse repositório. 
-                                              ( commit = salvar aqui ).
      Existem empresas aonde se padronizam os salvamentos ( Commit ), por exemplo de sintaxe: 
-                                    git commit -m "código-da-task/Tipo-da-task/Breve comentário"
Feature = Recurso
Bug = Erro
      Exemplo aplicado agora:       git commit -m "Feature/Tela-login Inicio "

Por resumo de tudo o Git é o versionador de códigos, onde atraves de comandos possamos salvar versões dos nossos códigos e 
o GitHub é o repositório onde salvamos estas versões que o Git criou.
